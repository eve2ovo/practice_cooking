家常菜通用烹饪知识库（适用于结构化菜谱生成）
一、使用说明
本知识库用于支持家常菜生成。
目标不是直接给出某一道菜的唯一标准答案，而是为模型提供稳定的烹饪依据，使其能够根据用户输入的菜名、人数、口味、现有食材，生成合理、可执行的菜谱步骤、所需时间、火候建议和语音播报内容。
生成菜谱时应遵循以下原则：
1. 优先根据菜名判断主要烹饪方法。
2. 再结合主要食材类别，确定预处理、下锅顺序、火候和时长。
3. 若用户提供现有食材，应优先使用。
4. 若缺少必要辅料，可补充常见基础调料，如盐、油、葱、姜、蒜、生抽、料酒。
5. 若没有精确菜谱，可基于通用方法和食材规则进行保守生成，不可凭空生成明显不相关的菜品。
6. 菜谱步骤必须真实可操作，不可空泛。
7. 同一道菜的做法允许有家常差异，但整体逻辑必须合理。
 
二、通用烹饪流程规则
大多数家常菜可以抽象为以下流程：
1. 确定主食材与辅料
• 主食材决定主体做法。
• 辅料决定风味、颜色和口感层次。
2. 食材预处理
• 清洗、去皮、切块、切片、切丝、切丁。
• 肉类必要时腌制或焯水。
• 易氧化食材如土豆可切后泡水。
• 腥味较重食材如牛羊肉、部分海鲜需要去腥处理。
3. 准备调料
• 家常基础调味一般包括：盐、糖、生抽、老抽、料酒、蚝油、胡椒粉、淀粉、食用油。
• 清淡口味减少油盐。
• 香辣口味可加入干辣椒、辣椒面、豆瓣酱、辣椒酱等。
4. 按烹饪方法执行
• 炒、蒸、煮、煎、炖、烤、炸、凉拌、焖、烧等方法各有不同火候和顺序。
5. 调味与收尾
• 大多数家常菜在临近成熟时进行最终调味。
• 收汁、撒葱花、装盘属于常见收尾动作。
 
三、烹饪方法规则
 
3.1 炒
定义
炒是家常菜最常见的方法之一，特点是油量适中、加热快、成熟快，适合快手菜。
适用食材
• 鸡蛋
• 叶菜类
• 肉片、肉丝
• 土豆丝、青椒、豆芽、西红柿等
通用流程
1. 先备料再开火。
2. 热锅后下油。
3. 油热后先下不易熟或需要爆香的食材。
4. 易熟食材后下。
5. 快速翻炒，避免久焖。
6. 临出锅前调味。
火候规则
• 多数快炒菜：中火或中大火。
• 青菜类：大火快炒。
• 鸡蛋类：中火更稳，避免过老。
• 蒜末、葱花爆香时间短，避免糊。
时间规则
• 备料：2～5分钟
• 爆香：10～20秒
• 主体翻炒：1～5分钟
• 总时长通常：5～15分钟
常见错误
1. 油未热就下锅。
2. 食材一次性全倒入，导致出水。
3. 火太小，炒成焖菜。
4. 长时间不翻动。
5. 调味过早，导致出水严重。
生成建议
• 步骤应体现“热锅下油—下食材—翻炒—调味—出锅”。
• voiceText 要短促明确，如“锅热后倒油，先下蒜末爆香”。
 
3.2 蒸
定义
蒸是利用水蒸气加热使食材成熟，特点是油少、口感嫩、原味保留较好。
适用食材
• 鸡蛋羹
• 鱼
• 南瓜
• 茄子
• 排骨
• 部分面食和点心
通用流程
1. 食材处理后装盘或装碗。
2. 水烧开后上锅蒸。
3. 根据食材厚薄控制时长。
4. 蒸好后再淋汁或撒配料。
火候规则
• 以中火到大火将水烧开。
• 蒸制阶段保持稳定沸腾。
• 不强调锅内直接火候，而强调“蒸锅持续有蒸汽”。
时间规则
• 鸡蛋羹：8～12分钟
• 鱼：8～15分钟
• 排骨：20～40分钟
• 根茎类块状食材：15～25分钟
常见错误
1. 冷水上锅时间判断不准。
2. 蒸太久导致口感老。
3. 容器太深导致受热不均。
4. 中途频繁开盖，影响成熟度。
生成建议
• 步骤要写明“水开后上锅”还是“冷水上锅”。
• 适合加入等待型步骤，如 type=wait。
 
3.3 煮
定义
煮是将食材放入液体中加热至成熟，适合汤类、面类、青菜、丸子、饺子等。
适用食材
• 面条
• 饺子
• 汤类蔬菜
• 丸子
• 豆腐
• 部分肉片
通用流程
1. 先将水或汤底加热。
2. 按成熟难度顺序下食材。
3. 控制煮制时间，避免久煮。
4. 最后调味。
火候规则
• 烧开阶段大火。
• 煮制阶段中火或中大火。
• 汤类可中小火保持稳定。
时间规则
• 青菜：30秒～2分钟
• 面条：3～8分钟
• 饺子：5～10分钟
• 豆腐汤：5～10分钟
常见错误
1. 所有食材一起下锅。
2. 久煮导致青菜发黄、豆腐碎裂。
3. 调味太早影响口感。
4. 水量过少导致糊底。
生成建议
• 步骤里要体现“水开后下食材”。
• 若为汤类，建议体现“烧开—下料—调味—关火”。
 
3.4 煎
定义
煎是利用较少油脂使食材一面或两面受热上色成熟，特点是表面焦香。
适用食材
• 鸡蛋
• 豆腐
• 鱼
• 牛排
• 鸡胸肉
• 饺子锅贴
通用流程
1. 平底锅加热后下少量油。
2. 放入食材，不频繁翻动。
3. 一面定型上色后再翻面。
4. 必要时加盖或加少量水辅助成熟。
火候规则
• 多数以中火为主。
• 火过大会外焦里生。
• 豆腐、鱼等易碎食材更适合中火。
时间规则
• 荷包蛋：2～4分钟
• 豆腐：4～8分钟
• 鱼块：5～10分钟
• 鸡胸肉：6～10分钟
常见错误
1. 频繁翻动导致不成形。
2. 锅不热或油太少导致粘锅。
3. 火太大导致焦糊。
4. 食材太厚却不加盖辅助成熟。
生成建议
• 步骤里要体现“先定型再翻面”。
• 对易碎食材要写“轻轻翻面”。
 
3.5 炖
定义
炖是将食材长时间加热至软烂入味的方法，适合肉类、根茎类和汤菜。
适用食材
• 牛肉
• 排骨
• 鸡块
• 土豆
• 胡萝卜
• 白萝卜
通用流程
1. 肉类先焯水或煸炒去腥。
2. 加入调料翻匀。
3. 加足量热水。
4. 中小火长时间炖煮。
5. 根茎类可后下。
6. 最后收汁或调味。
火候规则
• 前期翻炒：中火
• 炖煮阶段：中小火
• 收汁阶段：中火
时间规则
• 普通鸡块炖煮：20～35分钟
• 排骨：30～50分钟
• 牛肉：40～90分钟
• 土豆后加后再炖：10～20分钟
常见错误
1. 牛肉等硬质肉类未去腥直接炖。
2. 中途频繁加冷水。
3. 土豆下锅过早炖烂。
4. 一直大火导致水干、肉不烂。
生成建议
• 若菜名带“炖、煲、焖”，优先考虑炖类/焖类逻辑。
• 对牛肉、排骨建议加入焯水步骤。
 
3.6 烤
定义
烤是通过烤箱、空气炸锅或明火热空气加热使食材成熟并形成焦香外层。
适用食材
• 鸡翅
• 五花肉
• 土豆块
• 南瓜
• 玉米
• 茄子
通用流程
1. 食材预处理并腌制或拌油。
2. 预热烤箱或设备。
3. 平铺食材。
4. 中途翻面或刷料。
5. 烤至表面金黄成熟。
火候规则
• 烤制不直接写 HIGH/MEDIUM/LOW 时，可转为“预热—烤制—翻面继续烤”逻辑。
• 若必须映射锅火，可统一近似为 MEDIUM。
时间规则
• 鸡翅：15～25分钟
• 土豆块：20～35分钟
• 茄子：15～25分钟
常见错误
1. 不预热。
2. 食材堆叠，受热不均。
3. 中途不翻面。
4. 调味过重导致表面过早焦黑。
生成建议
• 明确写“预热设备”。
• 可使用等待型步骤。
 
3.7 炸
定义
炸是将食材放入较多热油中快速加热成熟，特点是外酥里嫩。
适用食材
• 鸡翅
• 肉丸
• 薯条
• 茄盒
• 小酥肉
• 春卷
通用流程
1. 食材裹粉、挂糊或直接处理。
2. 热油至合适温度。
3. 分批下锅。
4. 炸至定型或金黄。
5. 捞出沥油。
6. 需要更酥可复炸。
火候规则
• 中火到中大火。
• 油温要稳，不宜过低。
• 复炸通常时间短、火稍大。
时间规则
• 小块食材：2～5分钟
• 中块食材：4～8分钟
• 复炸：20～60秒
常见错误
1. 油温不够就下锅，导致吸油。
2. 一次放太多，油温骤降。
3. 长时间高温导致外焦里生。
4. 不沥油影响口感。
生成建议
• 步骤必须体现“分批下锅”。
• 炸类要提醒注意安全和防溅油。
 
3.8 凉拌
定义
凉拌是将食材处理成熟或直接生食后，加入调味汁拌匀食用，特点是清爽快捷。
适用食材
• 黄瓜
• 木耳
• 菠菜
• 金针菇
• 豆腐丝
• 鸡丝
通用流程
1. 食材洗净，必要时焯水。
2. 沥干水分。
3. 调配凉拌汁。
4. 将食材与调料拌匀。
5. 可静置片刻更入味。
火候规则
• 凉拌本身不强调锅火。
• 若需焯水，通常先大火烧开，再短时间焯烫。
时间规则
• 焯水：30秒～3分钟
• 拌制：1～3分钟
• 静置入味：5～10分钟
常见错误
1. 食材水分太多，导致味道淡。
2. 焯水过久，口感发软。
3. 拌得太早，出水严重。
4. 调料比例失衡。
生成建议
• 若为凉拌菜，时间应偏短。
• 适合清淡、酸辣、开胃等口味表达。
 
3.9 焖
定义
焖是先简单翻炒，再加少量液体并加盖，使食材在封闭环境中慢慢成熟入味。
适用食材
• 鸡翅
• 茄子
• 土豆
• 豆腐
• 五花肉
• 米饭类
通用流程
1. 食材先煎或炒出香味。
2. 加入调味和少量液体。
3. 盖盖焖煮。
4. 开盖收汁。
火候规则
• 前段翻炒：中火
• 焖制阶段：中小火
• 收汁阶段：中火
时间规则
• 叶菜或软质食材：3～8分钟
• 鸡翅、土豆等：10～20分钟
常见错误
1. 液体过多变成煮。
2. 不加盖，无法形成焖制效果。
3. 火太大，汁液很快烧干。
4. 一直不开盖检查，导致糊底。
生成建议
• 焖类步骤需体现“加盖焖几分钟”。
 
3.10 烧
定义
烧通常指先处理食材，再加入调味和液体继续加热至入味，介于炒和炖之间，常见于红烧、家常烧。
适用食材
• 鸡翅
• 茄子
• 豆腐
• 鱼块
• 五花肉
• 土豆块
通用流程
1. 先煎、先炒或先炸定型。
2. 加入酱油、料酒等调味。
3. 加适量水或汤。
4. 中火烧至入味。
5. 收汁后出锅。
火候规则
• 定型阶段：中火
• 烧制阶段：中小火
• 收汁阶段：中火
时间规则
• 软质食材：5～12分钟
• 鸡翅、鱼块：10～20分钟
• 五花肉等：20分钟以上
常见错误
1. 一开始加太多水，失去“烧”的浓郁感。
2. 不先定型，食材易散。
3. 过度收汁导致发咸。
4. 火太急，外面入色但内部不入味。
生成建议
• “红烧”类通常要体现上色、烧入味、收汁。
 
四、食材处理规则
 
4.1 鸡蛋
1. 鸡蛋成熟很快。
2. 火过大、时间过长容易变老。
3. 炒鸡蛋适合中火。
4. 可加少许清水提升嫩度。
5. 蛋液类菜肴对火候敏感。
 
4.2 牛肉
1. 牛肉纤维较粗，适合炖、煮、烧，也可切薄片快炒。
2. 炖牛肉常先焯水去血沫。
3. 长时间加热宜用中小火。
4. 若快炒，宜逆纹切薄片并提前腌制。
5. 牛肉未软烂前不宜过度收汁。
 
4.3 猪肉
1. 猪肉可炒、煎、烧、炖，适应范围广。
2. 肉片快炒适合先腌制。
3. 五花肉适合烧、焖、煎。
4. 瘦肉久煮易柴。
5. 猪肉需确保熟透。
 
4.4 鸡肉
1. 鸡胸肉易老，适合快炒、煎、烤。
2. 鸡腿肉、鸡翅更适合烧、焖、烤。
3. 鸡块炖煮前可焯水去腥。
4. 鸡肉不宜久炒。
 
4.5 鱼类
1. 鱼肉细嫩，适合蒸、煎、烧。
2. 翻面时要轻，避免散。
3. 蒸鱼通常水开后上锅。
4. 煎鱼要先定型再翻面。
5. 腥味重时可配姜、葱、料酒。
 
4.6 虾类与海鲜
1. 成熟很快，不宜久煮久炒。
2. 适合快炒、蒸、煮。
3. 过度加热会变老、缩水。
4. 调味宜清爽，突出鲜味。
 
4.7 豆腐
1. 豆腐易碎，翻动要轻。
2. 可煎、煮、烧、炖。
3. 煎豆腐适合先沥水。
4. 嫩豆腐更适合煮汤或蒸。
5. 老豆腐更适合煎、烧。
 
4.8 土豆
1. 土豆可炒、炖、煎、烤、炸。
2. 切丝后可泡水去淀粉。
3. 炖煮时不宜过早下锅，否则易过烂。
4. 炸制前需擦干表面水分。
5. 土豆块通常比叶菜成熟更慢。
 
4.9 叶菜类
1. 成熟很快。
2. 适合大火快炒或短时间焯水。
3. 久炒会出水、发黄。
4. 调味不宜过重。
 
4.10 根茎类蔬菜
1. 如胡萝卜、白萝卜、土豆等，通常成熟时间较长。
2. 适合炖、烧、蒸、煮。
3. 切块越大，成熟越慢。
4. 可在炖肉类时后段加入。
 
五、口味适配规则
5.1 清淡
1. 减少油、盐、辣。
2. 突出食材本味。
3. 避免重色重酱。
5.2 家常
1. 咸淡适中。
2. 常用基础调味。
3. 做法均衡，不极端。
5.3 微辣
1. 可少量加入辣椒、辣椒面或辣椒酱。
2. 辣味点到为止，不盖住主料。
5.4 香辣
1. 辣味和香味更突出。
2. 可用干辣椒、蒜、花椒、豆瓣酱等。
3. 但不能完全盖住主食材。
5.5 下饭
1. 可略增强咸鲜和酱香。
2. 适合稍有汤汁或更浓郁的调味。
 
六、人数缩放规则
1. 1人份通常约为2人份的0.6倍。
2. 3人份通常约为2人份的1.5倍。
3. 4人份通常约为2人份的1.8～2倍。
4. 主料可按人数增加。
5. 调味料通常不需要严格倍增，应略小于主料增幅。
6. 烹饪时间通常不会完全按人数线性增加。
7. 若人数增加较多，备料时间可增加，主体火候与单步时间只做适度调整。
 
七、步骤生成规则
1. timelineSteps 一般不少于4步。
2. 常见步骤包括：
• 准备食材
• 预处理
• 热锅/加热
• 主体烹饪
• 调味
• 收尾装盘
3. stepNo 从1开始递增。
4. timeOffset 应按前序 duration 累计递增。
5. duration 使用秒。
6. voiceText 比 instruction 更短、更适合播报。
7. confirm 步骤适用于：
• 热锅完成
• 水开后上锅
• 食材回锅前
• 收汁观察阶段
 
八、时间经验规则
8.1 常见备料时长
• 简单切菜：120～180秒
• 复杂处理肉类：180～420秒
8.2 常见爆香时长
• 葱姜蒜爆香：10～20秒
8.3 常见炒制时长
• 青菜：60～180秒
• 鸡蛋：60～120秒
• 肉丝肉片：120～240秒
8.4 常见蒸制时长
• 鸡蛋羹：480～720秒
• 鱼：480～900秒
8.5 常见炖煮时长
• 鸡块：1200～2100秒
• 排骨：1800～3000秒
• 牛肉：2400～5400秒
• 土豆后加：600～1200秒
8.6 常见煎制时长
• 荷包蛋：120～240秒
• 豆腐：240～480秒
8.7 常见凉拌时长
• 焯水+调味总计：300～600秒
 
九、生成兜底规则
1. 当没有命中精确菜谱时，应先根据菜名判断烹饪方法。
2. 再根据主食材类别补足合理步骤。
3. 若仍不确定，输出保守家常做法，不可替换成完全不同的菜。
4. 用户输入的菜名优先级最高。
5. dishName 必须与用户输入一致，不可随意更换。
6. 若知识不足，也只能围绕该菜名做最小补全。
7. 不得把“土豆炖牛肉”生成成“番茄炒蛋”或其他无关菜品。
 
十、输出约束建议
生成结果应尽量满足：
1. ingredients 不少于3项。
2. prepTips 不少于2项。
3. timelineSteps 不少于4步。
4. description 1～2句话。
5. heatLevel 仅使用：
• HIGH
• MEDIUM
• LOW
• OFF
6. type 仅使用：
• auto
• wait
• confirm
7. difficulty 仅使用：
• EASY
• MEDIUM
• HARD
